# freight-forwarding

General multi-modal freight forwarding plugin. Covers the workflow of a neutral
forwarder's desk operator — read the RFQ in the inbox, calculate the cargo,
quote the shipper, draft the transport document, run the customs checklist.

Built around four skills and a practice profile that learns how your operation
works. Bilingual EN/RO out of the box; add other languages in the cold-start
interview.

---

## Skills

| Skill | Triggers on | Output |
|---|---|---|
| [`rfq-drafting`](skills/rfq-drafting/SKILL.md) | RFQ in inbound or outbound direction, quote request, partner enquiry | Draft email — Romanian for local, English for international |
| [`cargo-calculation`](skills/cargo-calculation/SKILL.md) | Dimensions and weights — anywhere a chargeable weight, CBM, freight ton, or LDM is needed | Summary table with all four metrics + dominant factor |
| [`transport-documents`](skills/transport-documents/SKILL.md) | CMR / AWB / HBL / MBL / Booking confirmation / Proforma / Packing list | Structured document, .docx via the `docx` skill if asked |
| [`customs-checklist`](skills/customs-checklist/SKILL.md) | Import / export clearance, HS code lookup, customs documents | Required documents list + indicative TARIC duty + VAT base |

## Commands

| Command | What it does |
|---|---|
| `/freight-forwarding:cold-start-interview` | First run — captures your company, lanes, escalations into `CLAUDE.md` |
| `/freight-forwarding:cargo-calc` | Direct invocation of the cargo calculation skill |
| `/freight-forwarding:rfq-draft` | Direct invocation of the RFQ drafting skill |
| `/freight-forwarding:transport-doc` | Direct invocation of the document generator |
| `/freight-forwarding:customs-checklist` | Direct invocation of the customs checklist |

You don't have to use the slash commands. The skill descriptions are tuned to
trigger on natural-language phrasing the way an operator actually types
("calculează chargeable pe 4 colete 60x40x30 18kg" / "draft an RFQ to the
Frankfurt agent for 3 pallets dap"). Slash commands are there when you want
deterministic invocation.

## Agents

| Agent | Trigger | What it does |
|---|---|---|
| [`rfq-inbox-watcher`](agents/rfq-inbox-watcher.md) | New email in connected inbox tagged as RFQ | Classifies, calls `rfq-drafting`, holds for approval per Authority Matrix |

The agent consults [`../docs/authority-matrix.md`](../docs/authority-matrix.md)
before sending. Default is *do not send autonomously above €1K, do not
autonomously handle charter / OBC / dangerous goods*. Loosen in your practice
profile after you've seen it run.

## Practice profile

Stored at:

```
~/.claude/plugins/config/claude-for-freight/freight-forwarding/CLAUDE.md
```

Generated by `/freight-forwarding:cold-start-interview` and edited by you
afterward. Contains:

- Company identity (legal name, EORI, default address, signing details)
- Default incoterms when shipper doesn't specify
- Reference number format (`DCS-YYYY-NNNN`, `ACME-25-####`, etc.)
- Lane preferences and trusted agents per lane
- Escalation thresholds (per the Authority Matrix)
- Currency and unit conventions (EUR/USD/GBP; cm/inches; kg/lbs)
- IATA volumetric divisor (default 6000; some couriers use 5000)
- Standard trailer dimensions for LDM (default 2.4m × 13.6m)

Every skill reads this file before doing anything else. If a skill output
references your company's details, those come from the profile. If a
calculation uses a divisor, the divisor comes from the profile.

## MCP connectors

Skills work fine standalone — they'll produce text and ask you to copy-paste.
With MCP connectors active, the workflow shortens significantly. See
[`.mcp.json`](.mcp.json) for the connector configuration.

Recommended (all optional):

- **Gmail / Outlook** — read RFQ threads, draft replies in the live inbox
- **Google Drive / OneDrive** — save generated transport documents to the
  shipment folder structure
- **`cargo.one` / `WebCargo`** — pull live air rates during RFQ drafting
- **Your TMS** — record shipment master data after booking confirmation
  (CargoWise, Riege Scope, StarCargo connectors at [`/connectors`](../connectors/))

## Customization

Three places to customize, in order of how often you'll use them:

1. **Practice profile** (`CLAUDE.md`) — most changes go here. Lane preferences,
   default incoterms, agent network, escalation thresholds.

2. **Skill body** (`skills/*/SKILL.md`) — when you want a workflow change
   that's structural, not just a different default. Add a step, remove a step,
   change a template. Fork-and-modify is encouraged.

3. **Plugin metadata** (`.claude-plugin/plugin.json`) — only when you're
   building a derivative plugin for your own org. Rename, re-version, change
   the maintainer.

## Why these four skills and not twelve

A general-purpose plugin is most useful when it covers the 80% of desk
operator activity that's the same regardless of cargo type. RFQ, calculation,
transport document, customs checklist — those four cover the bulk of every
shipment.

When the shipment is *not* general — AOG, charter, OBC, DG, automotive
line-stop, cold-chain pharma — you want a specialized plugin that encodes
the workflow for that mode, not a general plugin trying to handle everything.
See [`../README.md`](../README.md) for the roadmap of specialized plugins.

The principle: this plugin is a strong baseline. The specialized plugins
are the upgrades for specific corners. Install both; they don't conflict.

## Disclaimer

This plugin produces drafts for a qualified operator's review. Cargo
calculations show the formula and inputs. Customs duty estimates show the
TARIC code and warn that the licensed declarant carries final responsibility.
Transport documents include placeholders that the operator must verify before
issuance.

Output is not a binding quote, not a filed declaration, not a substitute
for licensed customs broker, DGSA, or carrier consultation. A qualified
forwarder reviews and takes professional responsibility for anything that
leaves the desk.
